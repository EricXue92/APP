# Let's Tennis - 网球约球 iOS App 设计文档

## 概述

**Let's Tennis** 是一款面向网球爱好者的 iOS 约球应用，帮助水平相近的球友找到对手、约定球场、组织比赛。首期上线城市为香港，后续扩展至北京、上海、杭州、广州、深圳、武汉、成都、天津、沈阳。

**技术栈：** SwiftUI (iOS) + FastAPI (Python 后端) + PostgreSQL + Redis

**多语言：** 简体中文、繁体中文、英文

---

## 1. 用户系统

### 1.1 登录方式（4种）

- 手机号 + 验证码
- 微信一键登录
- 用户名 + 密码（需邮箱认证激活）
- Google 账号登录

**登录持久化：** JWT Token 存入 iOS Keychain，自动刷新。用户无需重复登录，除非主动退出。

### 1.2 注册流程

1. 选择登录方式并完成认证
2. 基本资料：昵称、头像、**性别（必填）**、所在城市
3. 网球水平设置：**自测问卷**（5-8 道选择题自动计算 NTRP）**或 直接输入**（如 3.0+、3.5-、2.5、4.0）
4. 完善资料（可选）：常去球场、偏好时间、打球年限、运动照片

### 1.3 个人主页

- 头像 + 昵称 + 性别 + 城市
- **水平标签：** 如 "3.5 中级" / "3.5+" / "4.0 中高级"
- 信用分 + 赴约率
- 打球统计（场次、常去球场、最近活跃）
- 收到的评价
- 隐私控制：可设置信息 公开 / 仅好友可见 / 隐藏

## 2. 约球功能

### 2.1 模式一：发布约球帖

发起人填写：

- 类型：单打 / 双打
- 日期 + 时间段
- 球场（地图搜索选择）
- 要求水平范围（如 3.0-4.0）
- 性别要求：仅男 / 仅女 / 不限
- 费用说明（AA 或免费）
- 备注（自带球、需要教练等）

**流程：** 发布 → 符合条件的用户看到 → 报名 → 发起人确认 → 约球成立 → 聊天沟通细节

**双打：** 需凑齐 2-4 人，发起人可自带搭档。

### 2.2 模式二：智能匹配

用户设置偏好：

- 可用时间段
- 偏好球场 / 可接受距离
- 单打 / 双打 / 都行
- 对手水平范围
- 性别偏好
- 费用默认 AA

**匹配权重（优先级从高到低）：**

1. NTRP ±0.5 优先
2. 时间重叠
3. 地理距离近
4. 信用分高优先
5. 性别匹配

**流程：** 系统推荐 → 双方确认 → 约球成立

## 3. 信用分体系

| 项目 | 分值 |
|------|------|
| 初始分数 | 80 分 |
| 满分上限 | 100 分 |
| 按时赴约 | +5 分/次 |
| 首次违规取消 | 不扣分，发出警告提醒 |
| 第2次起：24h 前取消 | -1 分 |
| 第2次起：12-24h 前取消 | -2 分 |
| 第2次起：2h 内取消/爽约 | -5 分 |
| 天气原因取消 | 不扣分（系统自动检测） |

**限制：** 信用分低于 60 分 → 限制发起约球。

## 4. 天气集成

- 约球详情页显示该时段天气（温度、降雨概率、风力、紫外线指数）
- 约球列表中天气图标快速预览
- **预警类型：**
  - 暴雨/台风 → 免责取消
  - 高温（35°C+）→ 提示中暑风险，建议改期或选择早晚时段
  - 强紫外线（UV 指数 8+）→ 提示防晒，建议避开正午
- 恶劣天气 → 推送通知 + 触发免责取消
- 数据源：和风天气 API（支持国内 + 香港）

## 5. 社交功能

### 5.1 关注与好友

- 单向关注（类似微博），无需对方同意
- 互相关注 = 好友
- 好友约球时收到优先通知

### 5.2 评价系统

- 打完球后双方互评（24h 内）
- 评分维度：球技水平、守时程度、球品态度（各 1-5 星）
- 可选文字评价
- 评价公开展示在个人主页
- 恶意评价可举报，管理员审核

### 5.3 打球记录与统计

- 自动记录每次约球（日期、球场、对手）
- 统计：总场次、本月场次、常去球场 Top3、最常搭档
- 个人日历视图查看打球历史

## 6. 聊天系统

- **私聊：** 约球确认后自动创建聊天会话
- **群聊：** 双打约球自动建群（2-4人）、赛事参赛者群聊
- 支持文字、图片、位置分享
- 未读消息角标 + 推送通知
- 聊天中可举报不当内容
- 敏感词过滤

## 7. 社区赛事

### 7.1 赛事类型

- 单打淘汰赛
- 双打淘汰赛
- 循环赛（小组赛）
- 阶梯赛（Ladder）— 挑战排名高于自己的选手

### 7.2 发起赛事

- 任何用户可发起（需信用分 ≥ 80）
- 设置：赛事名称、类型、水平范围、性别要求、人数上限
- 设置：日期范围、球场、报名费（可选，预留接口）
- 设置：赛制规则（几局几胜、抢七等）

### 7.3 赛事流程

1. 发起 → 报名（截止日期前）
2. 自动/手动抽签分组
3. 参赛者录入比分
4. 实时更新赛程表和排名
5. 赛事结束 → 生成成绩单 → 影响 NTRP 评级

### 7.4 赛事页面展示

- 赛程对阵图（淘汰赛树状图）
- 积分榜（循环赛/阶梯赛）
- 参赛者列表 + 水平
- 赛事群聊入口

---

## 8. 推送通知

### 8.1 通知类型

- **约球：** 有人报名、报名被确认/拒绝、约球被取消、约球前 2h 提醒、双打缺人
- **匹配：** 找到匹配、对方确认/拒绝
- **社交：** 新消息、有人关注、收到评价
- **赛事：** 报名开始/即将截止、对阵安排、比赛提醒
- **天气：** 恶劣天气预警（暴雨、高温、强紫外线）、免责取消通知

### 8.2 用户设置

- 按类型开关通知
- 免打扰时段
- 推送方式：APNs（iOS 原生推送）
- 通知语言跟随用户语言偏好

---

## 9. 支付与费用

**MVP 阶段：** 仅做费用展示（AA 金额参考），不走平台支付。球员线下自行 AA。

**预留接口：** 后续可接入 Apple Pay、微信支付、支付宝，支持球场费 AA 分摊和赛事报名费收取。

---

## 10. 安全与合规

### 10.1 举报系统

- 举报类型：爽约、骚扰、虚假信息、不当言论、其他
- 举报后 24h 内人工审核
- 处罚阶梯：警告 → 限制功能 → 封号

### 10.2 拉黑机制

- 拉黑后双方不可见、不匹配、不能私信
- 可在设置中管理黑名单

### 10.3 实名认证（可选）

- 头像真人审核（AI 识别 + 人工复审）
- 已认证用户显示认证标识，匹配优先级更高

### 10.4 法律合规

- 用户服务协议
- 隐私政策（符合《个人信息保护法》+ 香港 PDPO）
- 运动风险免责声明（首次约球前需确认）
- App Store 审核要求：举报功能、内容审核

## 12. 技术架构

### 12.1 系统架构

```
iOS App (SwiftUI) ←→ FastAPI Backend (Python) ←→ PostgreSQL + Redis
                                                    ↕
                                              第三方服务
                                              • Google Maps
                                              • 和风天气
                                              • APNs 推送
                                              • 微信开放平台
                                              • Google OAuth
                                              • SMS 短信服务
```

### 12.2 后端模块

- Auth（认证）：登录注册、JWT、OAuth
- User（用户）：资料管理、隐私设置
- Booking（约球）：发布、报名、确认、取消
- Matching（匹配）：智能匹配算法
- Chat（聊天）：WebSocket 实时通信
- Event（赛事）：创建、报名、抽签、比分、排名
- Credit（信用）：信用分计算、记录
- Weather（天气）：天气查询、预警
- Notification（通知）：APNs 推送

### 12.3 核心数据模型

- **User** — id, 昵称, 头像, 性别, 城市, NTRP等级, 信用分, 语言偏好, 注册方式
- **Booking** — id, 发起人, 类型(单打/双打), 球场, 时间, 水平范围, 性别要求, 费用, 状态
- **BookingParticipant** — booking_id, user_id, 角色, 状态
- **MatchPreference** — user_id, 可用时间, 偏好球场, 水平范围, 性别偏好
- **Court** — id, 名称, 地址, 经纬度, 城市, 类型(室内/室外)
- **ChatRoom / Message** — 房间, 消息, 发送者, 时间, 类型
- **Event** — id, 名称, 类型, 赛制, 水平范围, 性别, 人数, 日期, 球场, 状态
- **EventMatch** — event_id, 轮次, 选手A, 选手B, 比分, 状态
- **Review** — 评价人, 被评价人, booking_id, 球技分, 守时分, 球品分, 文字
- **Follow** — follower_id, following_id
- **Report / Block** — 举报人, 被举报人, 类型, 状态
- **CreditLog** — user_id, 变动分数, 原因, 时间

### 12.4 iOS 技术选型

- UI 框架：SwiftUI（iOS 16+）
- 架构模式：MVVM
- 网络层：URLSession + async/await
- WebSocket：URLSessionWebSocketTask
- 本地存储：SwiftData + Keychain（凭证）
- 地图：MapKit + Google Maps SDK
- 推送：APNs + UserNotifications
- 图片处理：Kingfisher（缓存 + 加载）
